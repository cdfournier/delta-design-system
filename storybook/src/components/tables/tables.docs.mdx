# Tables

Tables display structured data in rows and columns, making it easy to scan, compare, and understand relationships between data points. Tables are composed of a header row, body rows, and three cell variants.

## Anatomy

Tables consist of:

- **Table cell:** Individual data cell, available in three variants: text, header, and control.
- **Header row (`tr`):** A row of header cells with a distinct background color.
- **Header row group (`thead`):** Container for the header row.
- **Body row (`tr`):** A row of text or control cells.
- **Body row group (`tbody`):** Container for all body rows.
- **Table (assembled):** Complete component combining `thead` and `tbody` within a scroll wrapper.

## Variants

Tables have three cell variants that can be composed into any table structure.

- **Text:** Standard body cell for data content. Uses regular-weight text and appears in `tbody` rows.
- **Header:** Column heading cell. Uses bold-weight text and appears in the `thead` row.
- **Control:** Interactive cell containing a checkbox for row selection or binary inputs.

## States

Table interactivity is primarily expressed through the control cell checkbox.

- **Default:** Rows display bottom borders in `global/gray-medium`. Header row uses `section/background`.
- **Hover:** Body rows may show subtle highlight on hover.
- **Focus:** Interactive elements expose visible focus indicators.
- **Checked:** Checked control cell uses `brand/primary` for fill and border.
- **Indeterminate:** Select-all checkbox supports an indeterminate state when only some rows are selected.

**Light/Dark mode behavior:** Table and header backgrounds adapt to theme via `page/background` and `section/background`. Cell text inverts for readability. Row borders use mode-invariant `global/gray-medium`.

## Accessibility

### Keyboard navigation

All interactive elements within table cells must be keyboard accessible:

- **Tab / Shift + Tab:** Move focus through interactive controls.
- **Space:** Toggle a focused checkbox.
- **Enter:** Activate focused links or buttons.

### Screen readers

Use semantic table markup with `<thead>`, `<tbody>`, `<th scope="col">`, and a `<caption>` (visible or visually hidden) describing table content.

### ARIA attributes

- **aria-label:** Use on select-all checkbox (`Select all rows`).
- **aria-label (row checkboxes):** Describe row target (`Select row: Jane Smith`).
- **indeterminate (property):** Set via JavaScript for partial selection state.

### Focus management

All interactive elements must show visible focus indicators with logical tab order.

### Color contrast

Text must meet 4.5:1 minimum contrast against row backgrounds in both themes.

## Usage

### When to use

Use Tables when you need to:

- Display structured data with clearly defined rows and columns.
- Let users scan and compare values across attributes.
- Present data with meaningful column headers.
- Support row-level selection or actions via control cells.

### When not to use

Do not use Tables when:

- Displaying a single column of items (use a list instead).
- Arranging layout (use grid/flexbox instead).
- Data has no meaningful column relationships.
- A key/value pattern would be clearer (use Details).

### Content guidelines

- **Keep header labels short:** Aim for 1-3 words.
- **Align data consistently:** Keep alignment pattern per column.
- **Avoid empty cells:** Use placeholders such as `—`.
- **Limit column count:** Very wide tables are hard to scan.

### Placement

Wrap tables in a horizontal scroll container (`.table-wrapper`) at all viewport sizes for overflow handling.

### JavaScript

JavaScript is required to manage the indeterminate state of the select-all checkbox when only some rows are selected.

```js
// Manage select-all checkbox indeterminate state
const selectAll = document.querySelector('.table-checkbox[aria-label="Select all rows"]');
const rowCheckboxes = document.querySelectorAll('.table-checkbox:not([aria-label="Select all rows"])');

function updateSelectAll() {
  const checkedCount = [...rowCheckboxes].filter(cb => cb.checked).length;
  selectAll.checked = checkedCount === rowCheckboxes.length;
  selectAll.indeterminate = checkedCount > 0 && checkedCount < rowCheckboxes.length;
}

selectAll.addEventListener('change', () => {
  rowCheckboxes.forEach(cb => { cb.checked = selectAll.checked; });
});

rowCheckboxes.forEach(cb => {
  cb.addEventListener('change', updateSelectAll);
});
```
