# Banners

Banners are prominent content blocks for hero sections, promotional areas, and feature highlights. They combine imagery with structured text and optional actions.

## Anatomy

### Background image

- **Container:** Outer wrapper with border-radius and overflow clipping.
- **Background image:** Full-cover image layer filling the container.
- **Overlay (optional):** Semi-transparent dark layer for text readability.
- **Content:** Heading, body text, and optional CTA.

<Canvas of={ComponentStories.AnatomyFigure} />

### Split

- **Container:** Flex wrapper managing orientation and direction.
- **Image panel:** Dedicated visual panel.
- **Content panel:** Structured text panel with optional CTA.

<Canvas of={ComponentStories.AnatomySplitFigure} />

## Variants

### Background image

- **Alignment:** Left, center, right.
- **Overlay:** On/off.
- **Size:** Mobile or desktop minimum height behavior.

### Split

- **Orientation:** Vertical or horizontal.
- **Direction:** Default or reverse.

## Accessibility

- Ensure foreground text remains readable against image backgrounds.
- Use overlay when contrast is insufficient.
- Keep heading structure meaningful for screen readers.
- Do not encode critical meaning in imagery alone.

## Usage

### When to use

- Hero and feature sections.
- Campaign messaging.
- High-priority promotional placements.

### When not to use

- Dense informational layouts.
- Long-form text sections where readability is primary.
