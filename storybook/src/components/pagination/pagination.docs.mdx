# Pagination

Pagination provides visual navigation for carousels, slideshows, and multi-step flows. Dot indicators show the current position and total number of pages or slides.

## Anatomy

Pagination consists of:

- **Nav container:** Semantic `<nav>` wrapper that identifies the control to assistive technology.
- **List (`ul`):** Inner container providing the pill-shaped background, padding, and gap between indicators.
- **Indicator buttons:** Interactive elements rendered as `<button>` inside `<li>` items.

<Canvas of={ComponentStories.AnatomyFigure} />

## Variants

Pagination currently supports one indicator type. Additional types may be introduced in future releases.

- **Dot (default):** Small circular indicator. Selected state is filled; unselected indicators show border only.

## States

- **Default:** Unselected indicators display a circular border with transparent fill.
- **Selected:** One indicator is active at a time and rendered filled.
- **Hover:** Indicator scales to `1.2x` for pointer feedback.
- **Focus:** Keyboard focus uses a `2px` primary outline with `4px` offset.

## Accessibility

### Keyboard navigation

Pagination must be fully keyboard accessible:

- **Tab / Shift+Tab:** Move focus between indicator buttons.
- **Enter / Space:** Activate focused indicator.
- **Arrow keys (optional):** Can be used with roving tabindex implementations.

### Screen readers

Use `<nav aria-label="Pagination">` and button-based indicators in a semantic list structure.

### ARIA attributes

- **aria-label on nav:** Describes the landmark.
- **aria-label on each button:** Identifies destination (`Page 1`, `Slide 2 of 5`).
- **aria-current="page":** Applied only to currently selected indicator.

### Focus management

Use visible focus indicators with `outline` and `outline-offset`. Avoid moving focus programmatically unless associated content does so intentionally.

### Color contrast

Selected indicators meet non-text contrast requirements in both modes. Verify dark-mode contrast for `global/gray-medium` on the resolved background.

## Usage

### When to use

Use Pagination when you need to:

- Indicate position in a carousel or slideshow.
- Show progress through onboarding/setup flows.
- Navigate multi-step form sections.
- Display current slide in small fixed sets (typically 2-10).

### When not to use

Do not use Pagination when:

- Content exceeds 10 pages/steps.
- Users must jump to explicit page numbers.
- Pattern is for tabular/search result pagination.
- A stepper/progress bar better fits the interaction.

### Content guidelines

- **One indicator per page:** Dot count must match page/slide count.
- **Keep counts manageable:** Beyond 10 dots, use numbered patterns.
- **Write descriptive aria-labels:** Prefer context-rich labels like `Slide 3 of 5`.

### Placement

Position below controlled content, centered relative to content area, with consistent spacing.

## Code reference

### HTML

```html
<!-- Basic pagination -->
<nav aria-label="Pagination">
  <ul class="pagination">
    <li><button class="pagination-dot selected" aria-label="Page 1" aria-current="page"></button></li>
    <li><button class="pagination-dot" aria-label="Page 2"></button></li>
    <li><button class="pagination-dot" aria-label="Page 3"></button></li>
    <li><button class="pagination-dot" aria-label="Page 4"></button></li>
  </ul>
</nav>

<!-- Slideshow pagination with total count in labels -->
<nav aria-label="Slideshow navigation">
  <ul class="pagination">
    <li><button class="pagination-dot selected" aria-label="Slide 1 of 5" aria-current="page"></button></li>
    <li><button class="pagination-dot" aria-label="Slide 2 of 5"></button></li>
    <li><button class="pagination-dot" aria-label="Slide 3 of 5"></button></li>
    <li><button class="pagination-dot" aria-label="Slide 4 of 5"></button></li>
    <li><button class="pagination-dot" aria-label="Slide 5 of 5"></button></li>
  </ul>
</nav>
```

### JavaScript

```js
const dots = document.querySelectorAll('.pagination-dot');
let currentIndex = 0;

function updatePagination(index) {
  dots.forEach((dot) => {
    dot.classList.remove('selected');
    dot.removeAttribute('aria-current');
  });

  dots[index].classList.add('selected');
  dots[index].setAttribute('aria-current', 'page');
  currentIndex = index;
}

dots.forEach((dot, index) => {
  dot.addEventListener('click', () => {
    updatePagination(index);
    navigateToPage(index);
  });
});

function navigateToPage(index) {
  console.log(`Navigate to page ${index + 1}`);
}
```
