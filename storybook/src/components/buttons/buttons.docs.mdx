# Buttons

Buttons trigger actions or navigate users through interfaces. They communicate interactive affordance through visual weight, color, and feedback states.

## Anatomy

Buttons consist of:

- **Label:** Action-oriented text describing what the button does.
- **Container:** Background and border that define the clickable area and communicate variant/style.
- **Icon (optional):** Visual reinforcement placed before (left) or after (right) the label.

<Canvas of={ComponentStories.AnatomyFigure} />

## Variants

Buttons have two axes of variation: **variant** (color palette) and **style** (visual treatment).

### Variant

- **Primary (default):** Brand red palette for the most important action.
- **Secondary:** Brand teal palette for supporting actions.
- **White:** White foreground treatment for dark/colored backgrounds.

<Canvas of={ComponentStories.VariantsFigure} />

### Style

Regular text buttons support two styles:

- **Solid (default):** Filled background for strongest emphasis.
- **Transparent:** No fill or border for low-emphasis actions.

<Canvas of={ComponentStories.StyleFigure} />

### Icon-only buttons

Icon-only buttons support all three variants and all three styles (`solid`, `outline`, `transparent`).

- Outline style is used for icon-only buttons to provide middle visual weight.
- Icon-only buttons require accessible labeling (`aria-label` or visually hidden text).

<Canvas of={ComponentStories.IconOnlyFigure} />

## States

Button states provide feedback about interaction and availability.

### Default

Resting state before user interaction.

### Hover

Regular buttons invert appearance on hover. Transparent styles deepen tone. Icon-only outline styles invert between outline and fill.

<Canvas of={ComponentStories.HoverFigure} />

### Focus

Focus uses browser default focus ring. Do not suppress default focus behavior.

### Disabled

Disabled buttons use `status/disabled` (`#808080`) for background and border with white label text. Transparent disabled buttons use `status/disabled` text with no fill/border.

<Canvas of={ComponentStories.DisabledFigure} />

## Accessibility

### Keyboard navigation

- **Tab:** Moves focus to next button.
- **Shift + Tab:** Moves focus to previous button.
- **Enter:** Activates focused button.
- **Space:** Activates focused native `<button>`.

### Screen readers

Use semantic `<button>` for actions and `<a>` for navigation.

For icon-only buttons, prefer a visually hidden `<span>` label; `aria-label` is fallback.

### ARIA attributes

- `aria-disabled="true"` with `disabled` for disabled controls.
- `aria-label` only when no visible/hidden text label exists.
- `aria-pressed` for toggles.
- `aria-expanded` for disclosure controls.

### Focus management

Do not suppress default focus ring unless replaced with an accessible equivalent.

### Color contrast

Filled and outline states target WCAG AA contrast. Disabled components are intentionally reduced contrast as non-interactive controls.

## Usage

### When to use

- Primary actions (save, confirm, create)
- Secondary/supporting actions
- Form submission
- UI triggers (open modal, expand region, switch view)

### When not to use

- Page navigation (use links)
- Non-interactive decoration
- Inline text emphasis

### Content guidelines

- Lead with action verbs.
- Be specific.
- Keep labels short.
- Use sentence case.

### Placement

- In right-aligned action groups, place primary action on the right.
- In left-aligned content contexts, place primary action on the left.
- Stack full-width on mobile when needed.
- Use `md` spacing between grouped buttons.
- Separate destructive actions clearly.

## Code reference

### HTML

```html
<button type="button" class="btn btn-primary-solid">Save changes</button>
<a href="/page" class="btn btn-secondary-solid">View details</a>
<button type="button" class="btn btn-secondary-transparent">Learn more</button>
<button type="button" class="btn btn-primary-solid" disabled aria-disabled="true">Submit</button>

<button type="button" class="btn btn-primary-solid btn-icon" aria-label="Settings">
  <svg aria-hidden="true" focusable="false" width="16" height="16">...</svg>
</button>
```

### CSS

```css
.btn {
  display: inline-flex;
  align-items: center;
  justify-content: center;
  gap: var(--components-button-padding-gap);
  padding: var(--components-button-padding-t-b) var(--components-button-padding-l-r);
  border-radius: var(--components-button-border-radius);
  font-size: var(--components-button-font-size);
  line-height: var(--components-button-line-height);
  font-weight: 700;
  border: var(--components-button-border-width) solid transparent;
  transition: all 0.25s ease 0s;
}

.btn:disabled,
.btn[aria-disabled='true'] {
  background: var(--status-disabled);
  border-color: var(--status-disabled);
  color: var(--global-white);
}
```
