# Messages

Messages provide user notifications, alerts, and feedback in various contexts. This includes modal dialogs for detailed content, inline notifications for status updates, and toast messages for brief, non-intrusive information.

## Anatomy

### Message component

Message variants consist of:

- **Container:** Background wrapper with optional border-radius.
- **Content (Modal/Promo):** Content block with heading and body.
- **Message text (Error/Success/Confirmation):** Bold text label.
- **Close button:** Dismissal control (X icon).
- **Action buttons (Confirmation only):** Yes/No decision buttons.

### Toast component

Toast consists of:

- **Container:** Dark background wrapper.
- **Message text:** Brief notification content.
- **Close button (optional):** Dismissal control.

## Variants

Messages come in six variants covering full-screen dialogs, inline status notifications, and transient toasts.

### Modal

Full-viewport dialog for detailed content, forms, or multi-step workflows.

### Promo

Half-viewport promotional banner for announcements, offers, or feature highlights.

### Confirmation

### Error

Red inline notification bar for error messages and failure states.

### Success

Green inline notification bar for success confirmations and completion states.

### Toast

Simple, non-intrusive notification for brief informational messages that do not require immediate action.

## States

Messages support these interaction states:

- **Open:** Message is visible and interactive.
- **Closed:** Message is dismissed and hidden.
- **Hover:** Close buttons and confirmation actions show hover treatment.
- **Focus:** Interactive controls expose visible focus indicators.

Modal and Promo variants delegate main typography to Content. Error, Success, Confirmation, and Toast use bold message text.

## Accessibility

### Keyboard navigation

All message components must be fully keyboard accessible:

- **Tab / Shift+Tab:** Move focus through interactive controls.
- **Enter / Space:** Activate close and action buttons.
- **Escape:** Close dismissible messages and dialogs.

### Screen readers

Use `role="alert"` for Error/Confirmation, and `role="status"` with polite live regions for Success/Toast. Modal and Promo should use `<dialog>` or `role="dialog"` patterns.

### ARIA attributes

- **aria-labelledby:** Links to the message heading/text.
- **aria-describedby:** Links supplemental text.
- **aria-live:** `polite` for toast, `assertive` for critical alerts.
- **aria-modal:** `true` for blocking modal dialogs.
- **aria-label:** Required for icon-only close controls.

### Focus management

When a dialog opens, focus moves into it; when it closes, focus returns to the trigger.

### Color contrast

White text on Error, Success, and Toast backgrounds meets WCAG AA requirements.

## Usage

### When to use

Use Messages when:

- You need a modal for complex workflows.
- You want to announce promotional or instructional content.
- A destructive action needs explicit confirmation.
- An error or success result needs immediate feedback.
- A brief, low-priority notification is appropriate (toast).

### When not to use

Do not use Messages when:

- A simple inline confirmation is enough instead of a modal.
- Information is warning-level but not error-level.
- Message requires guaranteed user action but is presented as toast.
- You need to stack multiple modals.

### Content guidelines

- **Be concise:** Make copy easy to scan.
- **Be specific:** Explain what happened and what to do next.
- **Be actionable:** Include clear recovery or next-step language.
- **Use plain language:** Avoid jargon.
- **Match tone to severity:** Error content should feel more serious.

### Placement

Modal and Promo overlay the page. Inline variants appear near related content. Toast is fixed and non-obstructive.

## Code reference

### HTML

```html
<div class="message message-error" role="alert">
  <span class="message-text">An error occurred. Please try again.</span>
  <button class="message-close" aria-label="Dismiss error">...</button>
</div>

<div class="toast" role="status" aria-live="polite">
  <span class="toast-message">Item added to cart</span>
  <button class="toast-close" aria-label="Dismiss">...</button>
</div>
```

### CSS

```css
.message-error,
.message-success,
.message-confirmation {
  padding: 0.5rem;
  border-radius: 0.25rem;
  display: flex;
  align-items: center;
  gap: 0.5rem;
}

.message-text,
.toast-message {
  font-size: 0.75rem;
  line-height: 1.25rem;
  font-weight: 700;
}
```

### JavaScript

```js
function showToast(toastEl, duration = 5000) {
  toastEl.removeAttribute('hidden');
  setTimeout(() => {
    toastEl.setAttribute('hidden', '');
  }, duration);
}
```
