# Header

The Header provides primary site navigation, branding, and utility controls at the top of every page. It adapts between a compact mobile layout with a collapsible menu and a full widescreen layout with inline navigation and a color mode toggle.

## Anatomy

The Header has two layouts depending on viewport size. Both share a top bar with backdrop blur, a logo, and the color mode toggle, though the toggle moves to the mobile dropdown when open.

- **Top bar:** Container with backdrop blur effect, present in all variants.
- **Logo:** Brand mark, links to home.
- **Navigation (widescreen):** Horizontal list of text links displayed inline.
- **Color mode toggle (widescreen):** Sun icon, switch, and moon icon, visible inline in the top bar.
- **Menu button (mobile):** Hamburger icon when closed, close (X) icon when open.
- **Navigation dropdown (mobile open):** Vertical list of links with trailing arrow icons, plus the color mode toggle.

## Variants

The Header has two layout variants determined by viewport size, and a state variant for the mobile menu:

- **Widescreen:** Single horizontal top bar with all navigation visible inline; links have no trailing icons; color mode toggle sits at the right of the navigation row; no menu toggle button.
- **Mobile, closed:** Compact top bar showing only the logo and hamburger menu button; navigation and color mode toggle are hidden.
- **Mobile, open:** Top bar remains visible with the logo and a close button; navigation dropdown appears below with links and the color mode toggle at the bottom.

<Canvas of={ComponentStories.MobileVariantsFigure} />

## States

The Header's primary interactive state is the mobile menu open/closed toggle. Individual links follow the standard Link states.

- **Default (closed):** Mobile shows logo + hamburger only.
- **Open (mobile only):** Dropdown appears; hamburger changes to close (X).
- **Focus:** Logo, links, menu toggle, and color mode toggle should have visible focus.

## Accessibility

### Keyboard navigation

- **Tab:** Moves focus through logo link, navigation links, menu toggle, and color mode toggle.
- **Shift + Tab:** Reverses focus order.
- **Enter / Space:** Activates links and buttons, toggles mobile menu and color mode switch.
- **Escape:** Closes the mobile menu when open and returns focus to the menu button.

### Screen readers

Use `<header>` and wrap navigation in `<nav aria-label="Primary navigation">`. Use semantic `<ul>/<li>` structure for links. The menu toggle should always be a `<button>`.

### ARIA attributes

- `aria-label="Primary navigation"` on nav.
- `aria-label="Open menu"` / `aria-label="Close menu"` on menu toggle.
- `aria-expanded` on menu toggle.
- `aria-label="Toggle dark mode"` on the switch input.
- `aria-label="Home"` on logo link.

## Usage

### When to use

- Provide primary site navigation at the top of every page.
- Display brand identity consistently.
- Give quick access to top-level destinations.
- Include a persistent utility control such as color mode toggle.

### When not to use

- For secondary/contextual navigation.
- In focused modal or single-task flows where primary nav distracts.
- In embedded/iframe contexts where parent app already provides navigation.

### Content guidelines

- **Limit links:** 5–7 primary links maximum.
- **Use concise labels:** 1–2 words.
- **Prioritize by importance:** Most important first.
- **Maintain consistency:** Keep nav labels/order consistent across pages.

### Placement

The Header is always positioned at the top of the viewport and can be sticky or fixed.
