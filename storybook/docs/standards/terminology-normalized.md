# Terminology (Normalized)

Use these terms by default in Storybook docs and implementation notes.

If a Figma page explicitly uses a different label, mirror Figma on that page.

## Breakpoints

- `Mobile`
- `Tablet`
- `Widescreen`

Avoid `Desktop` unless the source page explicitly uses it.

## Modes

- `Light mode`
- `Dark mode`

## State Labels

- `Default`
- `Hover`
- `Active / Pressed`
- `Focus`
- `Disabled`
- `Invalid`
- `Valid`

## Writing Conventions

- Use American English (`color`, `behavior`).
- Use sentence case for body copy.
- Keep table property labels in CSS style (`font-size`, `line-height`, `border-radius`).
- Prefer semantic names over visual slang in guidance text.

## Scope Note

This file normalizes wording only. It does not override:

- Figma source content
- Token values
- Required section outlines
