Footers

The Footer provides site-wide navigation, branding, and utility controls at the bottom of every page. It adapts between stacked and horizontal layouts across breakpoints, and includes an integrated color mode toggle.

Anatomy

The Footer consists of:

  • Outer container: Full-width wrapper that provides outer padding and establishes the page-level background.
  • Group container: Inner container with a distinct background color and border-radius that visually groups all footer content.
  • Navigation section: A labeled <nav> containing an unordered list of text links.
  • Brand section: Logo mark and site heading, center-aligned. On widescreen, this is constrained to a maximum width and pushed to the right.
  • Color mode toggle: A Switch component flanked by sun and moon icons for toggling light and dark mode.
  • Fine print: Copyright or legal text, rendered below the group container in fine/small type.

Variants

The Footer has two layout variants driven by the data-size attribute, corresponding to the mobile and widescreen breakpoints.

  • Mobile: A vertically stacked layout for narrow viewports. Navigation links are center-aligned. The brand section sits below the nav, also center-aligned. Use this layout below the tablet breakpoint (768px).
  • Widescreen: A horizontal layout for wider viewports. Navigation links align to the left; the brand section is pushed to the right with margin-left: auto. Use this layout at the tablet breakpoint and above (768px+).

States

  • Default: The footer renders with a light gray group container background, black text for the heading and fine print, and red link text. The color mode toggle defaults to the light mode position.
  • Hover: Navigation links change color on hover: text/link (#FF5247) transitions to brand/secondary (#003442) in light mode; white transitions to brand/primary (#FF5247) in dark mode.
  • Focus: All interactive elements — navigation links and the color mode toggle — display visible focus indicators. Focus styles defer to the browser default or the global focus ring defined in the design system.

Typography

Navigation links use the small type token with bold weight. The heading uses the h4 type token. Fine print uses the fine token. All text tokens scale responsively between mobile and widescreen breakpoints.

Light/Dark mode behavior: Background and text colors adapt to the active theme. Navigation links change from red (#FF5247) in light mode to white (#FFFFFF) in dark mode for improved contrast. The switch thumb also changes from primary red (#FF5247) to white (#FFFFFF) in dark mode. In dark mode, heading and fine print text both resolve to white against the secondary-colored group container.

Accessibility

Keyboard navigation

The Footer must be fully keyboard accessible:

  • Tab: Moves focus through navigation links in order, then to the color mode toggle.
  • Shift + Tab: Moves focus in reverse order.
  • Enter: Activates the focused navigation link or toggle.
  • Space: Toggles the color mode switch when focused.

Screen readers

Use semantic HTML to ensure correct landmark and list behavior. The <footer> element is announced as a contentinfo landmark. Navigation must be wrapped in <nav aria-label="Footer navigation"> to distinguish it from other navigation landmarks on the page. Links should use descriptive text — avoid "Click here" or "Read more".

ARIA attributes

  • aria-label on <nav>: Always provide aria-label="Footer navigation" to disambiguate from the primary navigation.
  • aria-label on toggle: The color mode switch input must have aria-label="Toggle dark mode" or equivalent — icon-only controls are not accessible without a text label.
  • aria-checked on switch: Use aria-checked to communicate the current toggle state to assistive technology.
  • aria-hidden on decorative icons: Sun and moon icons are decorative and must have aria-hidden="true".

Focus management

All interactive elements must display a visible focus indicator. The footer does not trap focus or manage focus programmatically — standard tab order applies. Ensure focus ring styles meet 3:1 contrast against adjacent colors.

Color contrast

Navigation link text (#FF5247) against the light mode group container (#EEEEEE) must meet 4.5:1 for normal text — verify this passes. In dark mode, white text on #003442 meets the 4.5:1 requirement. Icon-only controls (sun/moon) require 3:1 contrast against their background. Test all states in both light and dark modes.

Usage

When to use

Use the Footer when you need to:

  • Provide secondary site-wide navigation at the bottom of a page.
  • Display copyright, legal notices, or persistent utility controls.
  • Offer a color mode toggle accessible from any page.
  • Reinforce brand identity with the logo and system name.

When not to use

Do not use the Footer when:

  • You need primary navigation — use the Header component instead.
  • You are building an embedded or widget context where a full-page footer is inappropriate — use a simpler fine-print or attribution element.
  • The page already has the Footer — there should only ever be one Footer per page.

Content guidelines

  • Limit navigation links: 4–6 links is ideal for scannability. The Footer is not a site map.
  • Prioritize utility links: About, Contact, Privacy, and Terms are the most common and expected footer links.
  • Keep fine print concise: Copyright year and essential legal text only. Long legal blocks belong on dedicated pages.
  • Keep the copyright year current: Update annually — stale years undermine credibility.
  • Label the toggle: Always include a visible or screen-reader-accessible label for the color mode toggle, not just icons.

Placement

The Footer sits at the very bottom of the page, outside the main content area. It spans the full viewport width. On pages with minimal content, use a sticky footer pattern to ensure it does not float in the middle of the viewport.

Best practices

Do

Keep links focused and minimal

Include only the most essential navigation items. The Footer supplements, not duplicates, the primary navigation.

Make the toggle accessible

Provide a text label for the color mode control — icons alone are not sufficient for keyboard or screen reader users.

Maintain brand consistency

Keep the logo, heading, and fine print in sync with the rest of the product. The Footer is often the last thing a user sees on a page.

Test both layout variants

Verify the mobile and widescreen layouts at their respective breakpoints on real devices, not just resized browser windows.

Don't

Don't overload with links

Too many links clutter the Footer and reduce its utility. If you need more links, consider a structured sitemap page.

Don't rely on color alone

Footer links use color to indicate interactivity, but also change on hover to provide additional feedback. Ensure sufficient contrast.

Don't place critical actions here

The Footer is a low-attention zone. Primary calls to action, sign-up prompts, or urgent messages belong in the main content area.

Don't let the copyright go stale

An outdated copyright year signals a neglected product. Automate it if possible.

Code reference

HTML

<!-- Footer -->
<footer>
  <div class="group">
    <div class="row">

      <!-- Navigation -->
      <nav aria-label="Footer navigation">
        <ul>
          <li><a href="/about" class="link">About</a></li>
          <li><a href="/contact" class="link">Contact</a></li>
          <li><a href="/privacy" class="link">Privacy</a></li>
          <li><a href="/terms" class="link">Terms</a></li>
        </ul>
      </nav>

      <!-- Brand -->
      <div class="brand">
        <div class="the-delta-design-system-mark">
          <svg class="logo" viewBox="0 0 1024 796" aria-hidden="true">
            <!-- Delta logo -->
          </svg>
          <h2>The Delta Design System</h2>
        </div>

        <!-- Color mode toggle -->
        <div class="toggle-color-mode">
          <svg class="icon" viewBox="0 0 16 16" fill="none" aria-hidden="true">
            <!-- Sun icon -->
          </svg>
          <input type="checkbox" role="switch" aria-label="Toggle dark mode">
          <svg class="icon" viewBox="0 0 16 16" fill="none" aria-hidden="true">
            <!-- Moon icon -->
          </svg>
        </div>
      </div>

    </div>

    <!-- Fine print -->
    <p class="copyright">&copy; 2026 The Delta Design System. All rights reserved.</p>
  </div>
</footer>

JavaScript

// Color mode toggle
const toggle = document.querySelector('.toggle-color-mode input[role="switch"]');

toggle.addEventListener('change', () => {
  const isDark = toggle.checked;
  toggle.setAttribute('aria-checked', String(isDark));
  document.documentElement.setAttribute('data-theme', isDark ? 'dark' : 'light');
});
