Layout

If the goal of content is to be discoverable and easy to absorb, then the layout should not complicate that process. The simplest, most efficient layout model is a fluid layout.

Currently, everything in the Delta Design System is built on a truly responsive design system using flexbox. While the <header> and <footer> and select other items feature their own framework, the content in the <main> section uses a series of nested boxes to create a page layout.

The parent container in the framework is a .row. Each .row can be assigned an additional class that defines how it's child's columns occupy the horizontal space of the row. On screens smaller than 48rem wide, all columns in the .row span the full width of the browser.

  • class="row wide" each .column child will span the full width of the browser window.
  • class="row fifty" each .column child will span 50% of the width of the browser window.
  • class="row golden left" the first .column will span 62% of the width of the browser window and the second .column will span 38%.
  • class="row golden right" the first .column will span 38% of the width of the browser window and the second .column will span 62%.
  • class="row thirds" each .column child will span approximately 33% of the width of the browser window.
  • class="row fourths" each .column child will span approximately 50% of the width of the browser window on screens wider than 48rem but smaller than 64rem. On screens wider than 64rem, each .column child will span approximately 25% of the width of the browser window.

Each element includes some variation of 1rem padding so that there should always be 2rems of padding around the parent. This padding is designed to play a key role (with typography) in the carefully considered and consistent vertical and horizontal spacing that is at the heart of the Delta Design System.

Just a quick note: the spacing of the "Golden Rows" is based on the Golden Ratio. There are some methods to the madness here.
