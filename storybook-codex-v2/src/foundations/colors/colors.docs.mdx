# Colors

The Delta Design System color palette is designed for clarity, accessibility, and consistency across light and dark modes. Every color serves a specific purpose and should be applied through tokens so meaning and contrast remain stable across components.

---

## Color philosophy

Colors communicate meaning, establish hierarchy, and reinforce brand identity. The system uses semantic naming so colors can be applied consistently while preserving their intended meaning across themes and interface contexts.

---

## The color system

The complete color system includes brand colors, global neutrals, status indicators, and overlay utilities. All colors are available as design tokens, and semantic tokens resolve those values appropriately across light and dark modes.

### Brand colors

Brand colors define Delta's visual identity and are used for key accents, link treatments, and high-emphasis UI moments. Brand tokens are primitive values and should not be repurposed for semantic status states.

{/* Figure placeholder: brand colors swatches */}

### Global colors

Global colors are the neutral foundation for text, surfaces, borders, and supporting UI elements. They provide a stable set of primitives that semantic tokens can reference in either mode.

{/* Figure placeholder: global colors swatches */}

### Status colors

Status colors communicate functional meaning such as disabled, invalid, valid, and read-only states. These colors should remain semantically distinct from brand colors and should be paired with text or iconography when conveying meaning.

{/* Figure placeholder: status colors swatches */}

### Overlay colors

Overlay colors are utility tokens used to improve legibility over imagery or layered content. They are defined as opacity-based overlays and should be applied consistently rather than recreated ad hoc.

The Delta Design System supports both light and dark color modes. Semantic tokens automatically resolve to the appropriate value for each mode, helping preserve contrast, readability, and consistent behavior across the interface.

## Accessibility

All color combinations should meet WCAG 2.1 Level AA contrast requirements for their intended use. The color system prioritizes accessibility without compromising hierarchy, legibility, or brand expression.

### Contrast requirements

- **Body text (16px):** Minimum `4.5:1` contrast ratio (WCAG AA)
- **Large text (18px+ or 14px+ bold):** Minimum `3:1` contrast ratio (WCAG AA)
- **UI components:** Minimum `3:1` contrast ratio (WCAG AA)

### Color and meaning

Status colors are never used alone. Error, success, warning, and other functional states must include both color and iconography or text so meaning is conveyed to all users, including users with color vision deficiencies.

## Usage

Use the following guidance to apply the color system consistently across products, components, and documentation.

### When to use semantic tokens

Use semantic tokens (for example `text/default`) instead of primitive or global tokens (for example `global/black`) whenever you are styling component UI. Semantic tokens provide theming support by resolving to the correct value for the active mode.

### Mode switching

Users should be able to override system preferences and manually select their preferred color mode. Respect user choice and persist that selection across sessions.

### Testing requirements

Always test designs in both light and dark modes to verify:

- All text meets minimum contrast requirements
- Status colors remain distinguishable
- Interactive elements are clearly visible
- Focus indicators have sufficient contrast

### Best practices

#### Do

- **Use semantic tokens**  
  Reference `text/default` instead of `global/black` so theming works correctly in light and dark modes.
- **Maintain consistent contrast**  
  Verify text meets minimum contrast requirements against its background in both light and dark modes.
- **Use status colors appropriately**  
  Use `status/invalid`, `status/valid`, and related tokens for functional states instead of brand colors.
- **Pair color with other indicators**  
  Combine status color with icons, labels, or patterns to convey meaning to all users.

#### Don't

- **Don't use hex values directly**  
  Reference design tokens instead of hard-coded values to preserve consistency and support mode switching.
- **Don't rely on color alone**  
  Color by itself is not sufficient for all users; pair color with text or iconography.
- **Don't use brand colors for status**  
  Keep brand colors for brand expression and use status colors for functional meaning.
- **Don't override contrast requirements**  
  Accessibility is not optional; do not reduce contrast below WCAG minimums.
