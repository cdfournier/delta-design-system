# Header

Headers provide primary navigation, brand presence, and quick utilities at the top of a layout.

## Anatomy

- `header-top`: frosted surface with logo and controls
- Desktop nav: inline list of links
- Mobile menu button: closed/open icon control
- Mobile dropdown panel: link list + color mode toggle
- Color mode row: sun icon, switch, moon icon

## Usage

### Do

- Use `desktop` for full nav and `mobile` with explicit `closed/open` menu states.
- Use `mobileState: interactive` in tooling previews when you want click-to-open/click-to-close behavior.
- Keep nav labels short and consistent.
- Reuse this header as a structural tool in templates and page compositions.

### Don't

- Do not overload this component with page-specific widgets.
- Do not break the mobile open/closed state model.
