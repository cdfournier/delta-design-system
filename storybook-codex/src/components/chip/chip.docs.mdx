# Chips

Chips are compact elements for selections, filters, and removable tags.

## Anatomy

- Container: rounded surface with variant color
- Label: short descriptive text
- Leading accessory (optional): avatar or icon
- Trailing accessory (optional): avatar, check, or dismiss button

## Accessibility

- Use `<button>` for interactive chips; use `<span>` for static chips.
- For selectable chips only, use `role="checkbox"` with `aria-checked`.
- Use descriptive `aria-label` text for dismiss buttons.
- Keep touch targets at least 44x44px in production contexts.

## Usage

### Do

- Keep labels concise and scannable.
- Group related chips and allow wrapping.
- Keep variant usage consistent inside a group.

### Don’t

- Do not use chips for primary CTA actions.
- Do not rely on color alone to convey state.
- Do not truncate long labels aggressively.
