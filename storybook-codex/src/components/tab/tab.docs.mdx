# Tabs

Tabs organize related content into separate panels and allow users to switch views without leaving the page.

## Anatomy

- `tab-button`: Trigger button with default or selected state
- `tablist`: Container for tab-button items (horizontal or vertical)
- `tab-panel`: Content panel with optional figure and heading/body content
- `tabs`: Assembled component containing tablist and tab panels

## Accessibility

- Uses semantic ARIA tab pattern: `role="tablist"`, `role="tab"`, `role="tabpanel"`
- Selected tab uses `aria-selected="true"` and `tabindex="0"`
- Inactive tabs use `aria-selected="false"` and `tabindex="-1"`
- Tab buttons support keyboard navigation: arrow keys, `Home`, and `End`
- Panels are hidden with the `hidden` attribute and `aria-hidden`
