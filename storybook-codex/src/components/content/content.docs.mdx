# Content

Content is a reusable molecular pattern for structured text blocks used in Cards, Banners, and other composed components.

## Anatomy

- Container: fluid-width wrapper with responsive padding
- Eyebrow (optional): small context label above heading
- Heading + body: primary text pair (`title+p` or `h+p`)
- CTA row (optional): off, link-only, one-button, or two-button action set
- Footer (optional): supporting fine text

## Usage

### Do

- Reuse this component inside larger molecules/organisms for text consistency.
- Pick `title+p` for hero-like prominence and `h+p` for standard sections.
- Enable optional slots only when the context needs them.

### Don't

- Do not mix both heading variants in the same content block.
- Do not add arbitrary spacing outside tokenized gaps.
- Do not duplicate CTA/footer logic in parent components.
