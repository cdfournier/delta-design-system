# Links

Links are navigation elements that move users between destinations while preserving clear hierarchy and accessibility.

## Anatomy

- Link text: underlined, descriptive label
- Arrow icon (optional): trailing directional affordance
- Color token: `text/link` by default, `text/hover` on hover/focus

## Accessibility

- Use semantic `<a>` elements for navigation destinations.
- Keep link text descriptive out of context (avoid “click here”).
- Keep underline visible for recognition and contrast.
- Keyboard support relies on native browser link behavior (`Tab`, `Enter`).

## Usage

### Do

- Use links for navigation and references.
- Keep labels concise and destination-oriented.
- Use optional arrow where directional emphasis helps.

### Don’t

- Do not use links as primary action buttons.
- Do not remove underlines from inline links.
- Do not use vague labels without context.
