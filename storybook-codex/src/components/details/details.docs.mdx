# Details

Details is an expandable disclosure component for progressive reveal of supplementary content.

## Anatomy

- Container: bordered wrapper with rounded corners
- Summary: always-visible trigger row
- Icon: plus when collapsed, minus when expanded
- Body: hidden content region revealed on expand

Details has no visual variants. It changes only by state and content.

## Accessibility

- Uses native `<details>` and `<summary>` semantics.
- Supports keyboard interaction by default (`Tab`, `Enter`, `Space`).
- Open/closed state is announced by assistive technologies.

## Usage

### Do

- Use for supplementary information and FAQ disclosures.
- Keep summary labels concise and descriptive.
- Group related disclosures in vertical lists.

### Don't

- Do not hide critical content required for task completion.
- Do not nest many disclosure levels.
- Do not replace primary navigation with details disclosures.
