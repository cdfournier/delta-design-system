# Buttons

Buttons trigger actions and guide users through key interface flows.

## Anatomy

- Label: action-oriented text
- Container: background, border, and shape
- Icon (optional): left, right, or icon-only
- Padding: tokenized clickable area

## Accessibility

- Use `<button>` for actions and `<a>` for navigation.
- Keep touch targets at least 44x44px for mobile.
- Include `disabled` and `aria-disabled="true"` when unavailable.
- For icon-only buttons, include hidden text labels for assistive tech.
- Use recognizable icons for icon-only controls (search, close, menu, settings).

## Usage Guidelines

### Do

- Lead with action verbs.
- Keep one primary button per local group.
- Use predictable placement and spacing.
- Keep labels concise (1-3 words when possible).

### Don't

- Do not use vague labels without context.
- Do not use buttons for page-to-page navigation.
- Do not rely on color alone for state meaning.
- Do not reduce tap/click targets below accessible size.
