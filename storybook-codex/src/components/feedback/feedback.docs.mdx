# Feedback

Feedback components visualize progress and measured values through `progress` and `meter` patterns.

## Anatomy

### Progress

- Label: context text above the bar
- Bar track: full range background
- Value fill: current progress segment
- Value text (optional): textual value output

### Meter

- Label: context text above the meter
- Meter shell: padded wrapper around bar
- Bar track: range background
- Value fill: semantic state fill (`optimum`, `low`, `high`)
- Min/Max labels (optional): range endpoints

## Accessibility

- Uses `role="progressbar"` for progress and `role="meter"` for meter track.
- Includes `aria-valuemin`, `aria-valuemax`, and `aria-valuenow`.
- Keep labels descriptive for context and assistive interpretation.

## Usage

### Do

- Use progress for completion/loading status.
- Use meter for bounded value quality/risk interpretation.
- Keep threshold semantics consistent across the product.

### Don't

- Do not use red/green semantics without textual context.
- Do not hide min/max meaning when using meter bounds.
- Do not animate value changes excessively.
