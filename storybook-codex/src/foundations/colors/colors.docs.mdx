# Colors

The Delta Design System color palette is designed for clarity, accessibility, and consistency across light and dark modes.

## Color Philosophy

Colors communicate meaning, establish hierarchy, and reinforce brand identity.
The system uses semantic naming to ensure colors are applied consistently across components.

## The Palette

The palette includes brand colors, global neutrals, status indicators, and overlay utilities.

## Accessibility

All color combinations should meet WCAG 2.1 AA for their intended use.

Contrast requirements:

  <thead>

    <tr><td>UI components and graphical objects</td><td>3:1</td></tr>

Color and meaning:

- Do not rely on color alone
- Pair status colors with text and/or iconography

## Usage

### When to use semantic tokens

Use semantic tokens like `text/default` instead of raw globals like `global/black`.
This keeps theming behavior correct across modes.

### Testing requirements

Always test in both light and dark modes to verify:

- text contrast remains compliant
- status colors remain distinguishable
- interactive states remain visible
- focus indicators remain clear

## Best Practices

### Do

- Use semantic tokens
- Maintain consistent contrast in both modes
- Use status colors only for status meaning
- Pair color with non-color indicators

### Don't

- Do not hardcode hex values in components
- Do not rely on color alone for meaning
- Do not use brand colors as status replacements
- Do not ignore mode-switch testing
