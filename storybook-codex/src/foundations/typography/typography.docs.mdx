# Typography

Typography expresses hierarchy and enables improved consumption of content.

## Typography Philosophy

Clear, readable typography is fundamental to communication.
The system uses two typefaces and a consistent scale to establish hierarchy and rhythm.

### Type scales

- Headlines (`h1` to `h4`): Minor Third direction
- Body copy: Major Third direction

### Relative units

Implementation uses rem units backed by px source tokens.
This preserves accessibility when users change browser font size.

## Usage

### Semantic HTML

- Never skip heading levels
- Use headings for structure, not visual size only
- Use one `h1` per page context

### Contrast and readability

- Body text should meet WCAG AA minimum contrast
- Maintain readable line length (target 45–75 characters)
- Use fine/small text sparingly for secondary and tertiary content

### Responsive sizing

Typography scales at widescreen breakpoints while preserving semantic structure.
Resize with CSS/tokens, not by changing semantic tags.

## Accessibility

### Do

- Follow heading hierarchy
- Maintain readable line length
- Use semantic markup (`strong`, `em`) for meaning
- Test typography at multiple viewport sizes

### Don't

- Do not skip heading levels
- Do not use headings only for styling
- Do not arbitrarily override system line heights
- Do not use fine text for primary body copy
