# Grid

A responsive grid system provides structure and consistency across layouts.

## Grid Philosophy

The Delta system separates architecture from presentation:

- Architecture: a 12-column model used for layout decisions
- Presentation: 4, 8, or 12 visible columns by breakpoint

The model stays consistent while the visual presentation adapts.

## Accessibility

### Visual hierarchy

Use grid structure to make priority and relationships clear.

### Reading order

- Keep DOM order aligned with visual order
- Avoid visual reordering that breaks logical reading flow
- Ensure stacked mobile layouts preserve intended sequence

## Usage

### Mobile-first approach

Start with mobile constraints, then progressively enhance for tablet and widescreen.

### In code

Implement with CSS Grid using a 12-column architecture and tokenized gutter:

```css
.grid {
  display: grid;
  grid-template-columns: repeat(12, 1fr);
  gap: var(--grid-gutter);
}
```

Use architecture-based spans (`3`, `4`, `6`, `8`, `12`) for predictable proportions.

## Usage Guidelines

### Do

- Use the grid consistently across components and pages
- Keep gutter width tokenized and consistent
- Center narrower content when readability benefits
- Validate responsive behavior at each breakpoint

### Don't

- Do not ignore mobile and tablet behavior
- Do not remove gutters and collapse visual breathing room
- Do not break reading order with CSS-only rearrangement
- Do not introduce ad hoc grid math outside the 12-column model
