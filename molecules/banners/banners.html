Banners

Banners are a key visual molecule in the Delta Design System. We designed banners for maximum flexibility. Here are the options you can control:

  • Photo background (if a photo background is wanted; this is an inline setting)
  • Overlay color and opacity (this is an inline setting)
  • Point-of-origin for photo background (center, center-bottom, center-top, left-center, left-bottom, left-top, right-center, right-bottom, or right-top)
  • Horizontal position of text (left or right)
  • Banner height (tall or short)

Tall Photo Banner

This is a tall photo banner, and the content is right-aligned using the '.right' class. Because the photo content is light, we apply the '.light' class to the banner, and this sets the content to black. Also note that the overlay opacity is set to zero.

Photo: London by Richard Tilney-Basset

Photo Banner Options

Let's try to break down some of these options and how they're coded.

  1. The parent relationship starts with <div class="banner">.
  2. We use the .tall class to set the banner to height: 90vh — the .short class sets the banner to height: 45vh. If you don't include one of these classes, the banner will be only as tall as the content requires.
  3. Use either a .left or .right class to control whether the banner's content aligns left or right horizontally (on screens wider than 48rem).
  4. The .bg-photo class sets up the banner for a background-photo.
  5. The .light or .dark class is used to tell the css the overall tone of the background photo. If the class is set to .light (for a light photo), the content is set to color: $black. If the class is set to .dark (for a dark photo), the content is set to color: $white.
  6. Next come the background photo alignment options:
  7. Lastly, we use an inline style attribute to set the background-image.
  8. At the next level down we have an <div class="overlay" style="opacity: 0;"> element. This inline element enables us to cast an overlay atop the background image to improve readability.
  9. The content of the banner is placed in a <div class="text"> container. This container always stretches to fit the full height of the banner and is always positioned in the vertical center of the banner with enough padding to make it legible (and consistent with the rest of the spacing in the design system). You can change the vertical alignment of the .text div by setting an inline justify-content style to the .banner.

Side by Side Banner

With this type of banner, you can create content with an image alongside (instead of used as a background) content.

Photo: House by Jonas Lavoie-Levesque

Side by Side Banner Details

  • The img comes first in the code stacking order. This way the img always appears on top of the .text on smaller screens.
  • Use a .left or .right class to control whether the .text <div> is placed on the left or right (horizontally) on screens wider than 48rem.
  • Use a class of .light or .dark to control the color of the content in the .text container.
  • Add a .tall or .short class to specify the min-height of the banner.
  • Set the background color to the banner using the classes defined in the Colors details of the design system.

Simple Solid Banner

Ever need something simple to call attention to some content? That's where this banner comes in.

Simple Solid Banner Details

  1. As with other banners, you can use a class of .light or .dark to control the color of the content in the .text container.
  2. You can also use a .left or .right class to control whether the .text <div> is placed on the left or right (horizontally) on screens wider than 48rem.
  3. You can use the .tall and .short classes to specify the min-height of these banners.
  4. Set the background color to the banner using the classes defined in the Colors details of the design system. For example, in the banner above we have used the .bg-lt-blue class.
  5. We've modified some of the button css depending on the background-color applied to improve the :hover and :focus states.
